Key Milestones in HTML History

  1. 1989:Tim Berners-Lee proposes the idea of the World Wide Web.
  2. 1991: The first version of HTML is released.
  3. 1995: HTML 2.0 is standardized by the IETF.
  4. 1997: HTML 3.2 introduces tables and applets.
  5. 1999: HTML 4.01 becomes the dominant version.
  6. 2000: XHTML 1.0 is introduced for stricter syntax.
  7. 2014: HTML5 is officially released, revolutionizing web development.
  8. 2016: HTML becomes a living standard, maintained by WHATWG.

CoreComponents

HTML elements are the fundamental building blocks of web pages, defining the structure and content. They typically consist of a start tag, content, and an end tag, although some elements are "empty" and do not contain content or an end tag. Here are some basic and commonly used HTML elements:

1.HTML: The root element that encloses all other HTML elements in a document.

2.HEAD: Contains meta-information about the HTML document, such as the title, character set, and links to stylesheets.

3.TITLE: Defines the title of the HTML document, displayed in the browser's title bar or tab.

3.BODY:Contains the visible content of the HTML document, including text, images, links, and other elements.

4.H1 to H6: Heading elements, used to define different levels of headings, with h1 being the most important and h6 the least.

5.P:Defines a paragraph of text.

6.A: Creates a hyperlink, allowing users to navigate to other web pages or resources.

7.img: Embeds an image into the document. This is an example of an empty element, requiring only a start tag with attributes like src (source) and alt (alternative text).

HTML elements can also have attributes, which provide additional information about the element, such as id (for unique identification), class (for applying CSS styles), and style (for inline CSS styling).

Dynamic Tags

Dynamic tags are HTML elements that are created, updated, or removed automatically while the webpage is running. They are not written permanently in the HTML file; instead, they appear or change because of JavaScript or web frameworks.

  • Behavior
    • Can be added to the page (new elements show up).
    • Can be modified (text, style, attributes, or structure can change).
    • Can be removed (element disappears without reloading the page).
Dynamic tags are normal HTML elements that are generated or updated in real-time on a webpage, making the page interactive and responsive to user actions or data.

Formatting Tags in HTML

Formatting tags are used in HTML to change the appearance of text — making it bold, italic, underlined, highlighted, small, etc. These tags don’t affect the meaning of the text but control how it looks in the browse

Common Formatting Tags
  • b=> Bold Text
  • Strong =>Storng importance(bold+semantic importance)
  • em => Emphasized text (italic + semantic importance).
  • u => Underlined text.
  • sup => Superscript (e.g., x², 10⁵).
  • sub => Subscript (e.g., H₂O, CO₂)

HTML Features

  • Simple and Easy – HTML is easy to learn and use.
  • Platform Independent – Works on any browser and operating system.
  • Supports Multimedia – Can display images, audio, and video
  • Hyperlinking – Allows linking between web pages.
  • Forms and Input – Provides text boxes, buttons, and other elements to collect user input.

    • HTML List

    1. Ordered List
      • Items are shown in a numbered order (1, 2, 3 or A, B, C, etc.).
      • Example: Steps in a process.
    2. Unordered List
      • Items are shown with bullets (●, ○, etc.).
      • Example: Shopping list
    3. Description List
      • Used for terms and descriptions (like a dictionary or FAQ).
      • Contains:
        • dt → Definition term (word/title).
        • dd → Definition description (explanation).

    HTML Tables

    • ables in HTML are used to arrange data into rows and columns.
    • A table is made up of rows and columns, where each cell holds data.
    • The main parts of a table are:
      • Table Header → represents the heading of a column.
      • Table Row → a horizontal line of cells.
      • Table Data → the actual content inside the cells.
    • Tables can also have a caption (title) and cells can be merged vertically or horizontally.
    • They are useful for showing structured information like timetables, marksheets, price lists, or reports.

    Iframe

    The iframe tag in HTML is used to show another webpage inside your own webpage. It works like a small box or window on the page where you can load content from another source. For example, you can use an iframe to display a YouTube video, a Google Map, advertisements, or even an entire website without leaving your page. The most important setting is the src attribute, which tells the browser which page or resource to display inside the frame. You can also adjust its width, height, and border to control how it looks. In short, iframe helps you display one web page inside another easily.

    Multimedia

    Multimedia in HTML refers to adding elements like images, audio, and video to web pages, making them more attractive and interactive. HTML provides different tags for multimedia: the img tag is used to display images, the audio tag is used to play sound or music, and the video tag is used to play video files. These tags support different file formats such as JPEG, PNG, MP3, MP4, and WebM, and they can include controls like play, pause, volume, and fullscreen for better user experience. HTML also allows embedding external multimedia content like YouTube videos or Google Maps using the iframe tag.

    Form Tags in HTML

    Forms in HTML are used to collect input from users such as text, passwords, selections, or files. They are commonly used for login pages, registrations, feedback, and searches. The main tag is:

    Common Tags

    • input → For different types of user input (text, password, email, checkbox, radio, button, file, etc.).
    • label → Adds a label/name for input fields.
    • textarea → Multi-line text box for long input.
    • select → Drop-down list.
    • option → Defines items inside a drop-down list.
    • button → Creates clickable buttons (submit, reset, or custom)
    • fieldset → Groups related form elements together.
    • legend → Title for a fieldset group.

    Key attribute

    • action → URL where form data is sent.
    • method → GET or POST (how data is sent).
    • name/id → Identifies the form.
